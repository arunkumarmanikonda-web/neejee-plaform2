# v23.40.26.1.2 — Three real bugs you caught (Sprint 26.1.2)

Builds on top of v23.40.26.1.1. Deploy this zip instead.

## Bug 1 — "AI name suggestions ignore the name I typed"
Before: typing "Banarsi silk saree" then opening AI Names returned random
crafts (Kanchipuram, Madhubani, Khadi, Ajrakh…).

Why: the suggester component never sent the typed name to the API. The
prompt asked for a name "based on craft+region+material" only.

Fix:
- `components/admin/AiNameSuggester.tsx` now forwards `name` + `description`
  to the API as a "workingName" anchor.
- `app/api/ai/content/route.ts` has a NEW prompt branch for `nameSuggestions`
  that REQUIRES every one of the 7 suggestions to refer to the SAME piece.
  "Banarsi silk saree" → 7 Banarasi-saree names with different angles
  (Heritage, Royal, Sensory, Craft-tech, Poetic, SEO, Trend).

## Bug 2 — "DRAFT WITH AI leaves Craft / Region / State / Cluster blank"
Before: only Material was filled because Craft/Region/State/Cluster were
explicitly excluded from the draftable list.

Fix in `app/api/admin/products/ai-draft-all/route.ts`:
- DRAFTABLE_FIELDS now includes `craft`, `region`, `state`, `cluster`.
- Directives added for each. AI infers them from the working name +
  description + category (e.g. "Banarsi silk saree" → craft=Banarasi,
  region=Varanasi, state=Uttar Pradesh, cluster=Madanpura weavers).
- Artisan name is NEVER auto-filled (per your direction).
- New prompt rule: "If a working name is provided, the entire draft must
  describe that same piece. Do not switch craft / category."

`AiDraftAllButton.tsx` Brief type widened; edit page now passes `state`.

## Bug 3 — "I don't see ✨ Resolve with AI / /admin/taxonomy/migrate is 404"
This is a deploy gap, not a code gap. The CategoryPicker, the migrate page
and the AI-resolve endpoint were ALL shipped in v26.1.1. The screenshot
you sent shows the OLD flat `<select>` dropdown — which proves the
v26.1.1 bundle was never extracted/deployed.

What v26.1.2 contains:
- everything from v26.1.1 (taxonomy picker + AI resolver + migrate page + bulk-import AI fallback)
- PLUS the three Bug fixes above

So this single deploy gets you current.
